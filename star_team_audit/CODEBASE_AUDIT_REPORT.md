## 🏗️ Architecture Analysis

### Intended Architecture (from Docs)
- **Pattern**: Client-Side SPA (React 18 + Vite)
- **State**: Local (Zustand + LocalStorage)
- **AI**: Direct Anthropic API integration (Client-side)
- **Privacy**: "Zero-Knowledge" (Data stays on device)

### Architectural Flaws (Design Level)
- **Data Durability**: Reliance on `localStorage` is extremely fragile. Browser cache clearing or using "Incognito" mode results in permanent data loss of clinical records.
- **Security/Cost**: Client-side API calls require exposing the API key to the browser (if hosted for multiple users) or forces every user to bring their own key (BYOK).
- **Safety**: No server-side monitoring means no ability to intervene in active suicide crises beyond automated text responses.

### Recommendation
1.  **Hybrid Architecture**: Introduce a lightweight backend (e.g., Supabase/Firebase) for **Encrypted Data Sync** to allow cross-device usage and backup.
2.  **Edge Function Proxy**: Move API calls to a serverless function to hide API keys from the client client.
