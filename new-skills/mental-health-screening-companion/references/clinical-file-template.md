# Clinical File Template — Per-User Session Tracking

A structured tracking artifact, surfaced to the user for their own continuity and transparency. **Not a HIPAA medical record.** Frame it as: "a running summary of what we've talked about, what we've tried, and what's next."

Create at the start of the first session. Update after every session. Review every 4–6 weeks.

---

## Update cadence

### After every session (always)

1. Add a new Session Note entry (Section 10).
2. Update current symptom severity (Section 3).
3. Update screener scores if administered (Section 4).
4. Update safety status and plan if any change (Section 8 / 8a).
5. Update progress tracking (Section 11).
6. Update next-session focus and reminders (Section 13).

### Every 4–6 weeks (review checkpoint)

1. Re-administer PHQ-9 and GAD-7.
2. Review overall trajectory.
3. Assess what's working / what isn't.
4. Adjust treatment plan, phase, or frequency if indicated.
5. Review referral follow-through.

### After any crisis contact

1. Log crisis as its own session entry with type "Crisis".
2. Update Section 8 risk level.
3. Refresh safety plan (Section 8a).
4. Increase session frequency if clinically warranted.
5. Document any referrals made, contacts encouraged (988, ER, trusted person).

---

## Surfacing the file to the user

- Offer to share the current state any time they ask ("Want to see where we are on the tracking file?").
- Let them correct or redact any section.
- Never treat the file as static truth — it is a living, user-correctable summary.
